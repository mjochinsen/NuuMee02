## 4. Styling Specifications

### 4.1 Color Palette

#### Primary Colors
- **Cyan**: #00f0d9 (rgb: 0, 240, 217)
- **Purple**: #3b1fe2 (rgb: 59, 31, 226)

#### Background Colors
- **Page Background**: #0a0f1a (very dark navy)
- **Card Background**: #1e2935 (dark navy)
- **Border/Divider**: #334155 (medium dark gray)

#### Text Colors
- **Primary Text**: #f1f5f9 (off-white)
- **Secondary Text**: #94a3b8 (gray-blue)
- **Tertiary Text**: #64748b (medium gray)

#### Status Colors
- **Success Green**: #05df72 (rgb: 5, 223, 114)
- **Error Red**: #ff6467 (rgb: 255, 100, 103)
- **Warning Yellow**: (not specified in data)

#### Gradients
- **Primary Gradient**: Linear gradient from cyan (#00f0d9) to purple (#3b1fe2)
  - Used for: Active buttons, CTAs, badges, highlights
  - Direction: Top to bottom (or diagonal)

### 4.2 Typography

**Font Family**: Arial (fallback: sans-serif)

#### Font Sizes
- **12px**: Small badges, labels, fine print
- **14px**: Secondary text, descriptions, button text
- **16px**: Body text, headings, navigation, list items
- **18px**: Emphasized text, taglines
- **20px**: Large descriptions, section intros

#### Font Weights
- **400 (Regular)**: All text (design uses single weight)

#### Line Heights
- **12px text**: ~18px line height (1.5x)
- **14px text**: ~21px line height (1.5x)
- **16px text**: 24px line height (1.5x)
- **18px text**: ~27px line height (1.5x)
- **20px text**: 28px line height (1.4x)

### 4.3 Spacing System

**Layout Spacing**:
- Section padding: 24px
- Card padding: 16-24px
- Item spacing: 8-12px
- Section gaps: 48-64px (vertical)

**Component Spacing**:
- Icon to text: 8-12px
- Button padding: 10-12px vertical, 16-20px horizontal
- Badge padding: 4-6px vertical, 8-12px horizontal

### 4.4 Border Radius
- **Small elements** (badges): 6px
- **Buttons**: 8-10px
- **Cards**: 12px
- **Large containers**: 16px

### 4.5 Shadows & Effects
- **Card elevation**: Subtle shadow (not explicitly defined, but implied)
- **Hover states**: Likely brightness/opacity changes
- **Gradient overlays**: 20% opacity for icon backgrounds

## 8. Accessibility Notes

### 8.1 Color Contrast
- White text on dark backgrounds: High contrast (WCAG AAA likely)
- Gray text on dark backgrounds: Medium contrast (check for WCAG AA)
- Colored badges: Check contrast ratios for accessibility

### 8.2 Interactive Elements
- Buttons should have clear focus states
- Links should be distinguishable from text
- Cards should indicate interactivity

### 8.3 Content Structure
- Proper heading hierarchy
- Alternative text for images (not visible in design, but needed)
- Clear labels for interactive elements

## 12. Technical Implementation Notes

### 12.1 Framework Recommendations
- **React/Next.js**: Component-based architecture ideal for card grids
- **Tailwind CSS**: Matches design system well
- **Framer Motion**: For animations and transitions
- **CSS Grid/Flexbox**: For layout systems

### 12.2 Component Reusability
- Create generic Card component with variants (success, error, use-case, testimonial)
- Reusable Badge component with type prop (success, error, info)
- Reusable Button component with variants (primary, secondary, outline)
- Icon system with consistent sizing

### 12.3 State Management
- Filter state (active category)
- Modal/overlay state (for expanded examples)
- User authentication state (credits, profile)
- Loading states for dynamic content

### 12.4 Performance Considerations
- Lazy load images (especially before/after comparisons)
- Optimize image formats (WebP with fallbacks)
- Code splitting by section
- Prefetch critical assets

## Notes for UI Planner

This design specification provides comprehensive details for implementing the Examples page. Key considerations:

1. **Component Reusability**: Many card types share similar structures - create a flexible Card component with variants
2. **Grid Flexibility**: The page uses 2, 3, and 4-column grids - use CSS Grid for consistency
3. **Dark Theme**: Entire page uses dark theme with careful attention to contrast ratios
4. **Gradient Accents**: The cyan-to-purple gradient is a brand signature - use consistently
5. **Status Indicators**: Clear success (green) and error (red) system throughout
6. **Content Structure**: Well-organized sections with clear hierarchy
7. **Responsive Design**: Grid layouts will need to collapse appropriately on smaller screens
8. **Performance**: Multiple images require optimization and lazy loading strategy
9. **Accessibility**: Ensure proper semantic HTML and ARIA labels for interactive elements
10. **Animation**: Smooth transitions enhance user experience - implement thoughtfully

The page successfully balances educational content (guides, tips) with social proof (use cases, testimonials) and clear CTAs, making it an effective showcase and conversion tool.
